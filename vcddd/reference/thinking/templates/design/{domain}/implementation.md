# 模板：design/{domain}/implementation.md

```
模板位置：vcddd/reference/templates/vcddd/design/{domain}/implementation.md
目标路径：docs/vcddd/design/{domain}/implementation.md
示例路径：docs/vcddd/design/order/implementation.md
          docs/vcddd/design/payment/implementation.md
```

**定位**：技术设计文档。描述代码层如何承载业务设计。代码必须与此一致，出现偏差以此为准修改代码，不以代码为准修改文档。

**写作要求**：
- 业务分支处理必须与 business.md 的命令处理路径一一对应，不能有遗漏
- 关键技术决策必须写明被否定的方案及原因，否则下次遇到同样问题会重蹈覆辙
- 幂等策略和事务边界是必填项，不可跳过
- 提前设计的优化必须说明背景和取舍，不能只写"做了什么"
