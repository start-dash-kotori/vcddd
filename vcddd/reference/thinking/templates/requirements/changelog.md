# 模板：requirements/changelog.md

```
模板位置：vcddd/reference/templates/vcddd/requirements/changelog.md
目标路径：docs/vcddd/requirements/changelog.md
```

**定位**：追加式变更记录。记录值得留存的内容：被推翻的方案及原因、踩过的坑、关键决策背景、思考变迁。未来按关键词检索，命中时作为决策参考。不是流水账，不记录琐碎的改动。

**写作要求**：
- 每条记录必须有"决策依据"，不只写"变了什么"
- 被否定的方案必须写明否定原因，否则失去参考价值
- 教训必须具体，不写"要注意边界"这种无操作意义的总结
- 新记录追加在文件顶部，保持时间倒序

---

```markdown
# 变更记录

<!-- 追加方式：新记录添加在文件顶部，保持时间倒序 -->

---

## {日期} — {变更标题}

**变更类型**：需求变更 / 边界调整 / 决策修正 / 踩坑记录 / 思考变迁

**背景**：
<!-- 必填。为什么会有这次变更，是什么事情触发了它。-->

**变更内容**：
<!-- 必填。变更了什么。原来是什么，现在是什么，涉及哪些域。-->

**决策依据**：
<!-- 必填。为什么这样决策。考虑了哪些方案，各方案的取舍是什么，最终选择的理由。
     被否定的方案：{方案} — 否定原因：{原因} -->

**影响范围**：
<!-- 这次变更对哪些域、哪些契约、哪些文档产生了影响。-->

**教训或思考**：
<!-- 选填，但踩坑记录必填。具体说明：踩了什么坑、为什么会踩、下次如何避免。-->

---
```
