# 模板：requirements/input.md

```
模板位置：vcddd/reference/templates/vcddd/requirements/input.md
目标路径：docs/vcddd/requirements/input.md
```

**定位**：用户用自然语言写下的业务意图、背景和设想。不要求准确，不要求结构化，只要求真实。模型会基于这个文件翻译成业务事实并与用户确认。

**写作要求**：
- 用自己的语言描述，不需要用 DDD 术语
- 写"想做什么"、"为什么要做"、"感觉有什么约束"即可
- 不确定的地方直接写不确定，不要猜测
- 可以随时补充，每次补充在原有内容基础上修改或追加

---

```markdown
# 需求描述

<!-- 最后更新：{日期} -->

## 背景与动机

<!-- 这个需求从哪里来？要解决什么问题？不解决会有什么影响？
     用自己的话写，不需要结构化。-->

## 想做的事

<!-- 描述你希望系统能做到什么。可以是用户故事、功能描述、流程描述，任何形式都可以。-->

## 感觉有的约束

<!-- 你觉得有哪些限制或规则？不确定的可以打问号。-->

## 不想做的事 / 明确不包含

<!-- 容易被误解为要做、但实际不打算做的部分。-->

## 开放问题

<!-- 你自己也没想清楚、需要讨论的部分。-->
```
