# 模板：design/{domain}/business.md

```
模板位置：vcddd/reference/templates/vcddd/design/{domain}/business.md
目标路径：docs/vcddd/design/{domain}/business.md
示例路径：docs/vcddd/design/order/business.md
          docs/vcddd/design/payment/business.md
```

**定位**：这个域的内部业务设计权威描述。从域的视角描述所有业务逻辑。代码必须与此一致，出现不一致时以此文档为准修改代码。

**写作要求**：
- 不变式必须写违反后果，没有后果描述的不变式是不完整的
- 状态机必须同时列出合法迁移和非法迁移，非法迁移是防止误操作的关键
- 命令处理路径必须覆盖所有失败分支，不只写正常路径
- 跨域协作必须说明本域消费了什么、产出了什么，以及失败时的处理方式

| {状态名} | | | |

### 合法迁移

```
待支付 ──[买家发起支付]──▶ 支付中
支付中 ──[支付已完成事件]──▶ 待发货
支付中 ──[支付失败事件]──▶ 待支付（重新等待支付）
待支付 ──[买家撤销 / 支付超时]──▶ 已取消
待发货 ──[卖家发货]──▶ 已发货
已发货 ──[买家确认收货]──▶ 已完成
```

### 非法迁移（明确禁止）

```
已完成 ──▶ 任何状态（终态不可逆）
已取消 ──▶ 任何状态（终态不可逆）
已发货 ──▶ 已取消（货已在途，不允许撤销）
```
