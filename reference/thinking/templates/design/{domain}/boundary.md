# 模板：design/{domain}/boundary.md

```
模板位置：vcddd/reference/templates/vcddd/design/{domain}/boundary.md
目标路径：docs/vcddd/design/{domain}/boundary.md
示例路径：docs/vcddd/design/order/boundary.md
          docs/vcddd/design/payment/boundary.md
```

**定位**：这个域对外的完整边界声明。外部协作者或大模型判断"能不能用这个域、怎么用"时读这个。不涉及内部实现细节。

**写作要求**：
- 每个命令必须写明前置条件，不写前置条件的命令是不完整的
- 事件必须说明它表达的是"已发生的事实"，不能描述成通知或回调
- "不做什么"必须写，且要写容易被误解的部分，不只写显而易见的边界
- 决策主权必须是可判断的业务问题，不是职责描述

### 对外事件

<!-- 必填。这个域会向外发布哪些事件。事件表达的是已经发生的业务事实，不是通知。

| 读模型 | 用途 | 包含的关键信息 | 不包含的信息（及原因） |

## 责任边界

### 决策主权

<!-- 必填。这个域对哪些业务问题拥有最终解释权。写成可判断的问题形式，不写职责描述。
     示例：
     - "这笔订单现在是否处于合法状态？" — 由本域决定
     - "这笔订单是否可以被撤销？" — 由本域决定（根据状态和时间窗口判断）
     - "撤销后退款金额是多少？" — 不由本域决定，由支付域决定 -->

### 不做什么

<!-- 必填。明确列出容易被误解为本域职责、但实际不属于本域的事情。
     格式：{事情} — 归属：{正确的域}
     示例：
     - 判断支付是否成功 — 归属：支付域
     - 决定退款金额 — 归属：支付域
     - 校验商品是否有货 — 归属：库存域（本域只消费库存域的结果） -->
